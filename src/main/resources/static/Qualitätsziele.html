Qualitätsziele

  • Ausfallsicherheit
    •  Die Anwendung muss 24 * 7 * 52 laufen.
    Sie darf nie ausfallen, da sonst Kunden verloren gehen.

  • Performance
    •  Eine Reservierung muss innerhalb von 2 min getätigt werden können.
    Jede Aktion auf einer User-Maske darf nicht länger als 500 ms dauern.

  • Erweiterbarkeit
    •  Neue Komponenten müssen leicht hinzuzufügen sein, ohne den Rest der Code-Basis anfassen zu müssen.

  • Bedienbarkeit
    •  Der Web-Client muss schnell verständlich sein.
    Das Backoffice muss innerhalt von 1 Tag Schulung erklärt werden können.
    Es soll möglich sein, eine Reservierung mittels Amazon's Alexa aufzugeben.
    Die Smart-Phone Anwendung muss sich gut ins Phone einfügen.
